# 火影 UI 美术字中译俄调研报告

- date: `2026-04-04`
- scope: `zh-CN -> ru-RU / Naruto / UI art / hard length budget`
- goal: `在 2.0x-2.5x 的俄文长度窗口内，尽量保留语义、火影 IP 识别度、以及俄罗斯手游玩家的阅读习惯`

## 1. 结论摘要

- 俄语 UI 美术字必须优先使用短名词、短动词和行业已接受的缩略写法；不要沿用书面化长结构。
- 面向俄罗斯手游用户时，`PvP / PvE / VIP / FAQ / ID / APK` 这类拉丁缩写可以保留；它们不是“翻译失败”，而是本地用户早已接受的界面写法。
- 正式系统/支付/UI 动作词仍要优先官方安全写法，如 `Магазин / Получить / Купить / Перейти / Рейтинг / Арена`；带明显玩家黑话色彩的词只适合活动图、推广图、或高语境强绑定的按钮。
- 对火影题材，专名必须稳定保留音译或既有常见译法，例如 `Наруто / Коноха / Ниндзюцу`。在长度紧张时，宁可压缩功能词，也不要改写 IP 专名。

## 2. 观察到的俄罗斯手游/UI 习惯

### 2.1 官方和商店页常见写法

- Google Play 俄语页常见短界面词：`Установить`、`Поделиться`、`Добавить в список желаний`、`Покупки в приложении`、`Есть реклама`。
- Steam 俄语页和游戏标签中，`PvP` 会直接保留拉丁写法，旁边再补充俄语解释；同时常见 `Игрок против игрока`、`Внутриигровые покупки` 这种正式界面术语。
- Google Play 上的俄语手游更新日志里，活动和赛季内容直接用 `Боевой пропуск`、`ивент-валюта`、`клан`、`награды` 等词，说明这些表达对俄罗斯移动游戏用户是自然的。

### 2.2 玩家语感和可接受短写

- 可直接接受的拉丁缩写：`PvP`、`PvE`、`VIP`、`FAQ`、`ID`、`APK`。
- 可接受的俄语游戏短写：`Крит.`、`ур.`、`нагр.`、`БП`。其中 `Крит.` 和 `ур.` 最安全；`нагр.` 只建议在极限长度场景使用；`БП` 只建议在强玩家语境下代表 `Боевой пропуск`。
- 可接受的玩家本土术语：`донат`、`гача`、`ивент`、`баннер`、`клан`。这些词可以进入活动图、召唤/卡池、推广图、促销位；不建议直接进入法律、计费说明、风格极正式的系统文案。

## 3. UI 美术字的压缩策略

### 3.1 优先顺序

1. 保留 IP 专名和功能核心词。
2. 优先把长名词短化，而不是压缩成生造缩写。
3. 优先用短动词命令式：`Получить / Купить / Перейти`。
4. 对已经被俄罗斯玩家普遍接受的外来词或缩写，允许保留：`PvP / VIP / FAQ / гача / донат`。
5. 如果压缩到 2.5x 内仍会损伤语义，直接进 review queue，不强行上线。

### 3.2 不建议的写法

- 不建议把普通按钮译成书面长句，如 `Ограниченное время`、`Таблица рейтинга`、`Первый платёж`。
- 不建议为了短而造本地用户不熟悉的缩写。
- 不建议把火影专名或核心术语用普通通用词替代，例如把 `忍术` 弱化成普通 `умение`。

## 4. 推荐短译表

| 中文 | 推荐俄文 | 说明 |
|---|---|---|
| 招募 | `Призыв` | 仅用于 summon/banner/gacha 语境；避免现有较弱的 `набор` |
| 领取 | `Получить` | 默认领取按钮 |
| 前往 | `Перейти` | 默认跳转按钮 |
| 购买 | `Купить` | 默认购买按钮 |
| 商店 | `Магазин` | 安全、短、通用 |
| 活动 | `Событие` | 比 `мероприятие` 更短、更适合 UI |
| 福利 | `Бонус` | 比 `льготы` 更符合手游语境 |
| 奖励 | `Награда` | 极端长度时才考虑 `Нагр.` |
| 排名 / 排行 / 排行榜 | `Рейтинг` | 统一短译 |
| 竞技 | `Арена` | 竞技/对抗面板常用短词 |
| 暴击 | `Крит` | 俄罗斯玩家高频短词 |
| 暴击伤害 | `Крит. урон` | 安全短写 |
| 充值 | `Пополнить` | 交易按钮优先正式短词；活动图文案可酌情用 `донат` |

## 5. 需要人工判断的高风险项

- `首充`：活动图可能接受 `1-й донат`，但正式付费按钮不宜直接黑话化；默认仍应走人工确认。
- `战力`：短成 `Сила` 会丢失“战斗力”语义，缩成 `БС` 又依赖玩家圈层，不建议自动化全量替换。
- `限时`：正式全写 `Ограниченное время` 过长，压成 `Лимит` 或 `Огр.` 都需要看具体 UI 位和上下文，不建议自动写死。
- `活动图宣传语`：如果用了 `ивент`、`донат`、`батлпасс/БП`，必须确认该位不是系统、合规、支付说明位。
